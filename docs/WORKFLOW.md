# Prompt 与工作流设计

## 总体工作流

1. 预处理：识别输入来源（文本/语音转写）、语言、敏感实体和对话角色。
2. 意图路由：判断用户要续写、改写、纠错、压缩还是综合处理。
3. 个性化：读取用户偏好标签，如“简洁、少用表情、偏正式”，不读取不必要的历史原文。
4. 生成：输出 3 个候选，并标注风格与置信度。
5. 安全审核：再次检测隐私、敏感内容和不当承诺。
6. 交互确认：用户点击候选回填，或一键脱敏后再发送。

## 系统 Prompt（示例）

```text
你是聊天输入助手。你的任务不是替用户做决定，而是帮助用户更清晰、自然、安全地表达。
输入：对话上下文、用户草稿、目标模式、用户表达偏好。
规则：
1. 保留用户原意，不编造事实，不擅自承诺时间、金额或结果。
2. 输出 3 个候选，优先短句，适合直接发送。
3. 续写要承接上下文；改写只改变语气；纠错要解释明显的语义修正。
4. 检测到隐私信息时返回 risk=true 和风险类型，但不要在日志中保存原文。
输出 JSON：{suggestions:[{text,style,reason}],risk:{hasRisk,type,action}}
```

## 语音纠错策略

先保留 ASR 原文，再结合上下文做候选纠错；只有当上下文一致性分数超过阈值时自动替换，否则展示“可能是……”供用户确认，避免过度纠错。

## 个性化策略

只存储可解释的偏好标签：平均句长、正式度、是否使用表情、常用称呼、常用结尾。用户可随时查看、修改和清空这些标签。
