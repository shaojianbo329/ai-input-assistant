# 用户测试与迭代记录

## 测试方案

对象：5 名有微信/钉钉高频沟通经验的学生或职场新人。

任务：分别完成“回复同事进度”“把口语改正式”“压缩长文本”“处理包含手机号的消息”四项任务。

## 第一轮观察

| 观察 | 证据 | 迭代决定 |
|---|---|---|
| 用户不确定按钮差异 | 2/5 人先后尝试多个按钮 | 将模式命名改成“接着说/改语气/变简洁/纠错” |
| 用户希望看到多个版本 | 4/5 人比较后再选择 | 固定展示 3 个候选 |
| 隐私提示需要更明确 | 3/5 人没有注意普通红色提示 | 使用橙色风险卡，并增加“一键脱敏” |
| 用户担心 AI 自作主张 | 2/5 人询问是否会自动发送 | 明确“点击候选才回填”，不自动发送 |

## 迭代后结论

原型从“一个生成按钮”改成“目标模式 + 多候选 + 安全确认”的三段式交互。下一轮应重点验证：建议采纳率、从输入到回填的时间、隐私提醒误报率，以及用户是否愿意开启个性化。

## 可复现记录

本轮为低保真可交互原型测试，样本量小，结论用于发现交互问题，不代表统计显著的产品效果结论。
