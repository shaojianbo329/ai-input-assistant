# 句伴产品方案

## 1. 项目定位

句伴是一款嵌入聊天输入框的 AI 表达辅助工具，核心价值是降低“想清楚但说不好”的沟通成本，同时在发送前提供隐私保护。

## 2. 用户痛点

| 痛点 | 典型场景 | 产品机会 |
|---|---|---|
| 回复成本高 | 不知道如何接话、怕回复不合适 | 基于上下文给出 3 个可选回复 |
| 语气不匹配 | 口语消息要发给老师、客户、领导 | 一键切换正式/礼貌/简洁 |
| 语音转写不准 | 同音字、口语省略导致语义偏差 | 结合上下文做语义纠错 |
| 信息太长 | 会议纪要、长段说明需要发在聊天里 | 自动提炼成短句 |
| 误发隐私 | 手机号、地址、身份证号未意识到风险 | 发送前提醒与脱敏 |

## 3. 竞品分析

| 竞品/替代方案 | 优势 | 空白点 | 句伴切入 |
|---|---|---|---|
| 输入法 AI 助手 | 入口近、覆盖面广 | 通常偏重改写，聊天上下文较弱 | 强化“接着说”和场景化建议 |
| 通用大模型对话 | 理解能力强 | 需要复制粘贴，打断聊天流 | 设计成发送前一步完成 |
| 邮件/办公润色工具 | 正式表达成熟 | 不适合即时聊天，使用成本高 | 提供简短、自然、可直接发送的结果 |

## 4. MVP 范围

首版聚焦 6 个能力：上下文续写、语气改写、纠错、个性化偏好、隐私提醒、长文本压缩。原型用 Mock 数据验证交互，模型接入作为下一阶段。

## 5. 产品原型说明

页面由“消息编辑器”和“AI 建议”两栏组成。用户输入文本后选择“接着说/改语气/变简洁/纠错”，点击建议即可回填编辑框。命中手机号、身份证号或地址时，建议区下方出现发送前提醒，并支持一键脱敏。

## 6. 关键取舍

- 默认给 3 个结果，兼顾选择空间与认知负担。
- 默认不自动发送，任何建议必须由用户确认。
- 个性化只学习表达偏好，不学习或长期保存敏感原文。

## 7. 效果评估指标

| 指标 | 定义 | MVP 目标 |
|---|---|---:|
| 建议采纳率 | 用户点击候选并回填的次数 / 建议展示次数 | ≥ 35% |
| 首次可用率 | 用户无需二次修改即可发送的建议占比 | ≥ 50% |
| 表达节省时间 | 使用助手与手动组织回复的平均耗时差 | ≥ 20% |
| 语义纠错准确率 | 人工标注正确的纠错结果 / 总纠错结果 | ≥ 90% |
| 隐私召回率 | 被识别出的敏感信息 / 全部敏感信息 | ≥ 95% |
| 隐私误报率 | 被误判为敏感信息的普通文本 / 普通文本总量 | ≤ 5% |
| 用户满意度 | 任务完成后 5 分制评分 | ≥ 4.0 |

评估时同时记录不同模式的表现，避免只看总体平均值；隐私相关指标优先于生成速度，且不以自动发送作为成功标准。
