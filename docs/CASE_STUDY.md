句伴：AI 智能输入助手作品集案例
## 项目背景
聊天输入是高频但高认知负担的场景：用户经常知道自己想表达什么，却需要花时间组织语气、纠正错字、压缩信息，甚至可能在发送前泄露隐私。句伴将 AI 能力放在“发送前最后一步”，降低表达成本，同时保留用户最终确认权。
## 我做了什么
- 从聊天场景拆解出续写、改写、纠错、压缩、个性化和隐私提醒六类需求。
- 设计“输入 → 意图识别 → 个性化 → 生成候选 → 安全检查 → 用户确认”的产品工作流。
- 完成可交互 Web MVP，支持语音识别能力检测与无能力时的演示兜底。
- 加入可解释分析链路，让用户看到当前识别的意图、表达偏好和风险状态。
- 通过 GitHub Pages 部署，形成可直接体验的产品链接。
## 关键产品判断
1. 不自动发送：AI 只负责建议，用户保留最终控制权。
2. 默认给 3 个候选：兼顾选择空间与决策成本。
3. 隐私检测前置：安全提醒优先于生成速度。
4. 个性化存偏好标签，不长期保存原始聊天内容。
## 作品集展示方式
面试演示时先输入：“这个方案我再看看吧，晚点给你”，点击“改语气”，展示正式表达；再点击“试试隐私提醒”，展示风险识别和一键脱敏；最后说明真实模型接入时只需要替换 `app.js` 中的 Mock 生成逻辑。
## 后续迭代
